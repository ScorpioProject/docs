## Definitions
- "We", "Us", "Our" refer to the ScorpioBot Development Team.
- "You", "Your" refer to the **Community Developers.**
- "Third Party Developers" refer to any developers who are not in contact with us and have not read this document.

## Community Developers
1. Every published resource must be submitted for verification before being published.
   - This includes both module releases, updates and patches.
   - No exceptions to this rule will be made.
2. Do not publicly call out any users or developers in the event of a dispute or conflict between two parties.
   - In case of any problems with other community members, please contact the ScorpioBot team.
3. You are not allowed to publish modules explicitly made by another third-party developer under any conditions.

## Submitting resources
1. Submitted resources may not contain any malicious code.
   - Malicious code is defined as inappropriate use of user's hardware or software access.
   - In case of high-risk function that has access to private data, the information needs to be clearly disclosed.
2. All submitted resources must consist primarily of your code.
   - You are allowed to use third-party or licensed code if permission is granted, but you are required to disclose its source.
   - You are not allowed to publish modules explicitly created by a third-party developer, regardless of your connection to them.
   - You are responsible for the safety and security of any third-party code you have included in your modules.
3. You are responsible for keeping your codebase clean and free of bugs for the duration of your holding the title of **Community Developer**. 
   - Any unmaintained modules must be strictly disclosed to be unmaintained and therefore possibly prone to bugs and/or vulnerabilities.
   - Any possible security issues must be fixed and disclosed to users of affected resource.

## Publishing resources
1. Add verified resource should be published on the [verified marketplace](/community/marketplaces/).
    - You are able to request verification of any platform you want to publish your resources on.
2. All threads for verified resources should include official branding for verified work.
    - You are not allowed to use verified branding on resources that were not verified by ScorpioBot team even if you are **Community Developer**.
3. All published resources need to clearly state that they are made for ScorpioBot along with link to the official GitHub repository of the project.
4. If your resource a dependency it needs to be disclosed to users before they obtain your module.

## Community relations
1. You are obligated to provide support for any published resource unless you clearly state otherwise.
2. You need treat every community member including developers and contributors with respect.
