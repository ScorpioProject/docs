ScorpioBot is built to be **efficient**, **scalable** and **easy to deploy**. You can run it both on a small managed hosting,
VPS, dedicated server or just a personal computer. The bot process is using minimal amount of resources so there shouldn't be
any problems meeting the minimal system requirements.

## Environment setup
To deploy ScorpioBot you need a [Node.js](https://nodejs.org/) runtime and package manager like [NPM](https://www.npmjs.com/)
or [Yarn](https://yarnpkg.com/). Currently, no other package managers are supported, but you can fill a GitHub Issue for that.

We recommend using the **latest LTS release** of Node.js and the **latest version of Yarn** for maximum compatibility,
but any version of Node.js grater than 20 and default NPM package manager will run the bot just fine.

## System requirements
To properly run ScorpioBot, you need at least **256 MB of system memory** and **512 MB of available storage space**.
If you are going to use multiple modules, we do recommend **512 MB of memory** and **1 GB of available storage space**.

## Managed hosting providers
If you prefer not to worry about software installation and server setup, we recommend a **managed hosting provider** tailored
for hosting DiscordJS apps. Some of our recommendations include [NovoNode](https://novonode.com/discord-bot-hosting) 
and [Sparked Host](https://sparkedhost.com/discord-bot-hosting) both starting at **1$ per month**.
